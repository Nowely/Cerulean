Task Creation from Template

Quickly create tasks with predefined structures and subtasks using templates.

Updated: February 17, 2026 | Interactive flow format

Preconditions

  • App running at http://localhost:3000
  • User has selected an active thread
  • Template library is populated with predefined task templates

Flow Steps

1
Action

Type "/template" in the message input to access template command.

Expected Result

Command palette appears showing "/template - Use a task template" option with an icon.

Panel simulates command menu entries for template actions.

Action

Click "From Template" option to open the template picker.

Expected Result

Template picker modal/drawer opens with title "Task Templates" and subtitle "Choose a template to quickly create a task". Displays 4 pre-configured templates: Bug Report (5 subtasks, "Standard bug report with reproduction steps"), Feature Request (4 subtasks, "New feature with design and implementation phases"), Sprint Task (4 subtasks, "Standard sprint task with full lifecycle"), Meeting Action Item (no subtasks, "Quick action item from a meeting"). Each template shows icon, title, description, and subtask count.

Key Observations

  • Multiple Access Points: Templates can be accessed via "/template" command or Quick Actions menu for flexibility
  • Template Categories: Templates are organized by common use cases (Bug Report, Feature Request, Sprint Task, Meeting Action Item)
  • Subtask Automation: Templates with subtasks automatically create parent task and all child tasks in a single action
  • Pre-filled Data: Templates include predefined titles, descriptions, status, priority, and subtask structures
  • Visual Indicators: Each template has a unique icon and clearly displays subtask count
  • Efficiency Gain: Templates save time by eliminating repetitive task setup for common workflows
  • One-click Creation: Selecting a template immediately creates the task(s) without additional form filling
  • Auto-assignment: Template-created tasks automatically assign the creator as the primary assignee
